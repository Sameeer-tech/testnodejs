## STEP 1: Prepare Code for GitHub (5 min)

Open PowerShell and run these commands:

```powershell
cd "c:\Aptech assignments\testnodejs"
git init
git add .
git commit -m "Chat app ready for deployment"
```

Then go to **https://github.com/new** and create a repo named `testnodejs`.

After creating, copy the HTTPS URL and run:

```powershell
git remote add origin https://github.com/YOUR_USERNAME/testnodejs.git

```

✅ Your code is now on GitHub!

## STEP 2: Deploy Backend on Render.com (10 min)

1. Go to **https://render.com** → Sign up with GitHub
2. Click "New +" → "Web Service"
3. Select your `testnodejs` repository
4. Fill in:
   - Name: `chat-app-backend`
   - Environment: `Node`
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance: `Free`

5. Add Environment Variables (one by one):

```
PORT = 5000
NODE_ENV = production
MONGODB_URI = mongodb+srv://uksameer2006_db_user:UF5BV8KgX3JZxjZd@test.lgqqyqm.mongodb.net/chat_app?retryWrites=true&w=majority
JWT_SECRET = your_super_secret_key_change_this_in_production
```

6. Click "Create Web Service"
7. Wait 2-3 minutes
8. **Copy your backend URL** (like: `https://chat-app-backend-xxxxx.onrender.com`)

✅ Backend is LIVE!

---

## STEP 3: Deploy Frontend on Vercel (5 min)

1. Go to **https://vercel.com** → Log in with GitHub
2. Click "Add New" → "Project"
3. Import your `testnodejs` repository
4. Configure:
   - Project Name: `chat-app-frontend`
   - Framework: `Create React App`
   - Root Directory: `client`

5. Add Environment Variable:

```
REACT_APP_API_URL = https://chat-app-backend-xxxxx.onrender.com
```
(Use YOUR backend URL from Step 2!)

6. Click "Deploy"
7. Wait 1-2 minutes
8. **Copy your frontend URL** (like: `https://chat-app-frontend.vercel.app`)

✅ Frontend is LIVE!

## 🆘 Quick Troubleshooting

**Backend URL?**
- Check Render dashboard → your service

**Frontend URL?**
- Check Vercel dashboard → your project

**Can't see messages?**
- Check browser console (F12)
- Verify backend URL in Vercel env variables

**MongoDB error?**
- Check env variable in Render is correct
- Check MongoDB Atlas Network Access
